# 自动账本 Roadmap

## P0 Harness 与安全

- 本地化 Supabase 配置，避免 key 上传 GitHub。
- 建立多 Agent 角色、脚本和质量门禁。
- 每次提交前运行密钥扫描、单元测试和 debug 构建。
- 增加 harness 脚本自测，覆盖运行记录、git 卫生和 schema 注释误判。（已加入 `scripts/harness/test-harness.sh`）

## P1 日常可用性

- 优化手动记账表单：分类选择、账户选择、备注、收入/退款/转账类型。（已支持类型和备注）
- 待确认账单纠错后确认：修改类型、金额、商户、分类、账户、备注后直接入账。（已支持）
- 待确认箱批量确认/忽略。（已支持首页全部待确认和明细当前筛选范围）
- 捕获通知后去重，避免同一通知重复入账。（已加入短时间同源同金额同商户去重）
- 增加常见银行、微信、支付宝、云闪付、外卖、电商样例解析测试。（已补云闪付、银行收入、千分位金额、实付优先）
- 降低营销通知误入账，过滤支付券/红包领取类文案。（已补支付立减券样例）

## P2 数据完整性

- Room schema 版本化和迁移测试。
- Supabase 增量拉取和 tombstone 删除同步，避免每次全量读取并保证多设备删除传播。
- 本地备份恢复：CSV/JSON 导入。
- 换机密钥导出和二维码展示。

## P3 统计体验

- 月度预算、分类预算、超支提醒。
- 日历视图、商户排行榜、账户趋势。（已修正默认本月统计口径）
- 自定义分类管理。
- 首页卡片可配置。

## P4 发布质量

- Release 签名配置本地化。
- APK 安装烟测脚本。
- 崩溃日志采集和用户可读错误提示。
